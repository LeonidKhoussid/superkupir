# Описание данных `data/places`

Набор нормализованных CSV для рекомендательной системы и гео-контекста. Кодировка **UTF-8 с BOM** (`utf-8-sig`), разделитель полей — **запятая**, много значений в одной ячейке — через **`;`**.

Файлы **`food.csv`**, **`hotels.csv`**, **`locations.csv`**, **`wines_main.csv`** подготовлены скриптом [`etls_pipline.py`](../../scripts/etls_pipline.py) из резервной копии `data — копия/` (очистка колонок + поля `city`, `type`). **`weather.csv`** — прогноз по городам/населённым пунктам (исходная схема без пайплайна `etls_pipline`).

---

## Общие поля (где есть)

| Колонка | Тип | Описание |
|---------|-----|----------|
| `city` | `str` | Населённый пункт из фиксированного списка, извлечённый из `address` (и для вин при необходимости из `source_location`). Пусто, если в тексте нет совпадения. |
| `type` | `str` | Вид датасета: `food`, `hotels`, `locations`, `wines_main` — совпадает с именем файла без `.csv`. |

**Допустимые значения `city` (подстрока в адресе):**  
`Южная Озереевка`, `Широкая Балка`, `Абрау-Дюрсо`, `Новороссийск`, `Мысхако`, `Федотовка`  
(при разборе длинные названия проверяются раньше коротких).

## `food.csv`

Рестораны и заведения общепита.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `place_id` | `int` (как текст в CSV) | Уникальный идентификатор точки в таблице еды. |
| `name` | `str` | Название заведения. |
| `address` | `str` | Полный адрес одной строкой. |
| `lat` | `float` | Широта WGS-84. |
| `lon` | `float` | Долгота WGS-84. |
| `place_type` | `str` | Тип заведения (например `ресторан`, `кафе`). |
| `cuisine_types` | `str` | Кухни и форматы через `;`. |
| `avg_bill_level` | `str` | Уровень среднего чека: `бесплатно`, `низкий`, `средний`, `высокий` и т.п. |
| `avg_bill_rub` | `int` | Средний чек в рублях (число). |
| `rating` | `float` | Оценка (обычно 0–5). |
| `review_count` | `int` | Число отзывов. |
| `description` | `str` | Краткое текстовое описание. |
| `city` | `str` | См. общие поля. |
| `type` | `str` | Константа `food`. |

---

## `hotels.csv`

Отели и объекты размещения (упрощённая схема после ETL).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `place_id` | `int` (как текст) | Уникальный id в таблице отелей. |
| `name` | `str` | Название объекта. |
| `address` | `str` | Адрес. |
| `lat` | `float` | Широта. |
| `lon` | `float` | Долгота. |
| `hotel_type` | `str` | Тип размещения: `отель`, `гостевой_дом`, `гостиница`, `апарт-отель`, `бутик-отель` и др. |
| `rating` | `float` | Рейтинг. |
| `tags` | `str` | Теги через `;` (русские ярлыки). |
| `description` | `str` | Описание. |
| `city` | `str` | См. общие поля. |
| `type` | `str` | Константа `hotels`. |

*Примечание:* цена за ночь, звёздность, булевы признаки и колонки с пропусками удалены на этапе ETL по правилам пайплайна.

## `locations.csv`

Точки интереса: парки, музеи, смотровые и т.д.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `place_id` | `int` (как текст) | Уникальный id локации. |
| `name` | `str` | Название. |
| `address` | `str` | Адрес. |
| `lat` | `float` | Широта. |
| `lon` | `float` | Долгота. |

| `location_type` | `str` | Тип локации: `парк`, `музей`, `смотровая_площадка`, `винодельня` и др. |
| `themes` | `str` | Темы через `;`. |
| `seasonality` | `str` | Сезон доступности: `круглый_год`, `зима`, `лето`, `не_зимой` и т.п. |
| `child_engagement_score` | `float` | Оценка вовлечённости детей (0–1). |
| `view_quality_score` | `float` | Оценка качества вида (0–1). |
| `rating` | `float` | Рейтинг. |
| `review_count` | `int` | Число отзывов. |
| `tags` | `str` | Теги через `;`. |
| `description` | `str` | Описание. |
| `city` | `str` | См. общие поля. |
| `type` | `str` | Константа `locations`. |

## `wines_main.csv`

Винодельни (источник — выгрузка каталога).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `wines_id` | `str` | Идентификатор записи (например `001`). |
| `name` | `str` | Название винодельни. |
| `source_location` | `str` | Регион/подпись источника (текст). |
| `card_url` | `str` | URL карточки на сайте-источнике. |
| `logo_url` | `str` | URL логотипа. |
| `lat` | `float` | Широта. |
| `lon` | `float` | Долгота. |
| `coordinates_raw` | `str` | Координаты как в источнике (текст). |
| `address` | `str` | Адрес или населённый пункт. |
| `city` | `str` | См. общие поля; может быть пустым, если ни `address`, ни `source_location` не содержат город из списка. |
| `type` | `str` | Константа `wines_main`. |

*Примечание:* в ETL остаются только колонки **без единой пустой ячейки** по всему файлу; состав колонок может меняться при обновлении сырья.

## `weather.csv`

Почасовой/по слотам прогноз для маршрутизации и фильтра «плохая погода».

| Колонка | Тип | Описание |
|---------|-----|----------|
| `date` | `str` (ISO date) | Дата `YYYY-MM-DD`. |
| `time` | `str` или `int` | Час слота (как в источнике, например `09`). |
| `city` | `str` | Город/пункт прогноза. |
| `weekday` | `str` | День недели по-русски. |
| `temp` | `int` | Температура, °C. |
| `weather_code` | `str` | Условное обозначение погоды (текст). |
| `is_bad_for_outdoor` | `bool` | `true` / `false` — нежелательно ли планировать активности на улице. |

---

## Связь с приложением

- Репозиторий читает места из [`app/config.py`](../../app/config.py) → `places_dir` / пути к `food.csv`, `hotels.csv` и т.д.; при переносе в продакшен путь можно указывать на `data/places_new`.
- Внутренние `place_id` в коде часто префиксируются (`food_1`, `hotel_1`, …); в этих CSV хранится **числовой фрагмент источника** в колонке `place_id` / `wines_id`.
