# Край Тур — фронтенд (хакатон)

Vite + React + TypeScript + Tailwind CSS. Только UI и клиентский флоу квиза, без бэкенда.

## Переменные окружения (фрагмент)

- **`VITE_YANDEX_MAPS_API_KEY`** — ключ JavaScript API Яндекс.Карт (секция `#discover`, маршруты).
- **`VITE_API_BASE_URL`** — полный origin бэкенда, если API не на том же хосте, что страница (опционально).
- **`VITE_API_PORT`** — порт бэка при авто-сборке URL (по умолчанию **3000**).

База URL для `fetch` задаётся в **`src/lib/apiBaseUrl.ts`**: при **`npm run dev`** клиент ходит на **`http://127.0.0.1:3000`**; при **`npm run preview`** и production-сборке — на **`http(s)://<хост страницы>:3000`**, чтобы браузер не блокировал запросы с публичного IP на **localhost** (Chrome Private Network Access). Если открываете **`vite dev --host`** с телефона, задайте **`VITE_API_BASE_URL`**. Подробнее — **`memory_frontend.md`**.

- **`/routes/:id/panorama`**: карта и Street View встраиваются **iframe** с публичными URL Google Maps — **отдельный ключ API в `.env` не нужен**.

## Запуск

```bash
cd front
npm install
npm run dev
```

Сборка:

```bash
npm run build
npm run preview
```

## Маршруты

- `/` — лендинг (hero + блок «Как работает сервис»)
- `/quiz/1` … `/quiz/6` — шаги квиза
- `/quiz/done` — финальный экран с моковым итогом (ответы из Zustand)

## Ассеты

Референсные изображения экранов лежат в `src/assets/` и `src/assets/quiz/`. Иллюстрации справа на шагах квиза — кроп полноэкранных макетов (`object-position` + масштаб), отдельных вырезов карточек нет.

## Примечания по совпадению с макетом

- Шрифт: **Comfortaa** (Google Fonts) как ближайший к округлой геометрии из макета.
- Декоративные «петли» по краям квиза — упрощённые SVG, не вектор из Figma.
- Плавающие 3D-объекты на hero — упрощённые SVG-заглушки; в макете — отдельные 3D-рендеры.
- На шагах 3–6 в макете варианты ответов совпадают с блоком длительности — воспроизведено как на референсе.
