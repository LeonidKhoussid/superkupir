# Край Тур — хакатонный проект

Веб-приложение для планирования путешествий по Краснодарскому краю: подбор мест, сборка маршрутов, публикация впечатлений, карты и панорамы Яндекса.

## Стек технологий

### Фронтенд (`front/`)

| Технология | Назначение |
|------------|------------|
| **Vite 8** | Сборка и dev-сервер |
| **React 19** + **TypeScript** | UI |
| **React Router 7** | Маршрутизация SPA |
| **Tailwind CSS 4** (`@tailwindcss/vite`) | Стили |
| **Zustand** | Состояние (квиз, корзина маршрута, auth) |
| **@pbe/react-yandex-maps** | Карты и панорамы (API 2.1) |
| **ESLint 9** + **typescript-eslint** | Линтинг |

### Бэкенд (`back/`)

| Технология | Назначение |
|------------|------------|
| **Node.js** + **TypeScript** | Runtime и типизация |
| **Express 5** | HTTP API |
| **PostgreSQL** (`pg`) | База данных |
| **Zod** | Валидация и конфиг |
| **jsonwebtoken** + **bcryptjs** | JWT и пароли |
| **swagger-ui-express** | Документация API в браузере |

## Описание и функционал

- **Лендинг** — hero с медиа, карусель мест с лайками и комментариями, блок «Как работает сервис», навигация и вход.
- **Квиз** — пошаговый опрос предпочтений; итог и сборка маршрута через API.
- **Каталог мест (`/places`)** — список и подборка, конструктор маршрута, рекомендации по якорю и сезону; на широких экранах — split-view с панелью деталей места.
- **Карточка места (`/places/:id`)** — описание, фото, координаты, добавление в маршрут.
- **Мои туры (`/myroutes`)** — сохранённые маршруты и маршруты с совместным доступом.
- **Маршрут (`/routes/:id`)** — просмотр и редактирование остановок, карта маршрута, шаринг, панорамы.
- **Впечатления (`/impressions`)** — лента постов, публикация из своего маршрута.
- **Авторизация** — регистрация, вход, JWT для защищённых запросов.

Публичное описание контрактов и сценариев также отражено в `front/memory_frontend.md` и `back/memory_backend.md`.

## Ссылки на фронтенд

- **Код:** каталог [`front/`](./front/)
- **Локальный запуск** (порт по умолчанию Vite — **5173**):

  ```bash
  cd front
  npm install
  npm run dev
  ```

  Приложение: **http://localhost:5173**

- **Переменные окружения** (создайте `front/.env` при необходимости):
  - **`VITE_API_BASE_URL`** — базовый URL API (без завершающего `/`). В коде запасной вариант — `http://localhost:3000`; **согласуйте порт с бэкендом** (см. ниже).
  - **`VITE_YANDEX_MAPS_API_KEY`** — ключ JS API Яндекс.Карт (карты на лендинге, маршрутах, панораме).
  - **`VITE_PUBLIC_APP_URL`** — публичный URL фронта для ссылок «поделиться» (опционально).

## Ссылки на Swagger / OpenAPI

Документация поднимается вместе с бэкендом.

- **Интерактивная документация (Swagger UI):**  
  **http://localhost:4000/api-docs**  
  (если не переопределён `PORT` в `.env` бэкенда)

- **Спецификация OpenAPI (JSON):**  
  **http://localhost:4000/openapi.json**

Исходник спецификации: [`back/src/swagger/openapi-spec.ts`](./back/src/swagger/openapi-spec.ts).

## Запуск бэкенда

```bash
cd back
npm install
# задать DATABASE_URL, JWT_SECRET (≥32 символов) и при необходимости PORT в .env
npm run dev
```

По умолчанию сервер слушает порт **4000** (`back/src/config/env.ts`). Укажите во фронте:

```env
VITE_API_BASE_URL=http://localhost:4000
```
